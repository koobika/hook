◆ Raw

base::AutoBuffer koobika::hook::network::protocol::http::Response::Raw

HTTP RAW response data. Writing to this field will bypass StatusCode, ReasonPhrase, Headers and Body properties while sending the response data. It should only be used in case of writing customized response data (bypassing default provided data preparation mechanism). In other words, write to the Raw field if you want to deeply control the data being sent to the client (e.g.: custom writers). It is strongly recommended to use default provided mechanisms (through StatusCode, ReasonPhrase, Headers and Body fields).

Example

server->Handle("/foo/bar", [server](const Request& req, Response& res) {
res.Raw.Write(
"HTTP/1.1 200 OK\r\nServer: Example\r\nContent-Type: "
"text/plain\r\nDate: Mon, 02 Jan 2006 15:04:05 "
"GMT\r\nContent-Length: 13\r\n\r\nHello, World!");
});
